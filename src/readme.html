Introduction

This is a table top audio based magic the gathering game. No rules are enforced, that is up to the user, however, this application provides a platform for people to search for cards, build decks, and play magic as if you were playing with actual cards on the table. The goal of this project is to emulate playing a game of magic with actual cards, not to enforce rules. Magic the Gathering is a very complex game, and I try my best to distance my self from enforcing rules do to the complexity, which would take a multi person team to build and maintain.

The game's interface should be easy to learn, but also includes a command line type interface for more advanced users. I strived to take the best of both worlds, to provide the user with the most ideal interface, and still keeping the project reasonably small. I used python to build this project, mainly because of the easy access to various open source libraries, and the ability to make the game cross platform.

This program uses a library called mtgsdk which leverages the mtg API

http://magicthegathering.io/

This library pulls down all of the magic card data, and it is stored into a local sqlite3 database for easy access. The user can update the database by launching a an executable in the database folder, which will update the database. There is also an executable for rebuilding the entire database, which will take some time. The executables use a command line interface for output.

to provide cross platform speech output, mtg audio uses accessible_output2. This provides speech support for a veriety of screenreaders across windows, Mac, and linux. The most common screenreaders are accounted for,
JAWS, NVDA, SAPI, Voiceover, and ESpeak.
For a comprehensive list of supported screenreaders, go to:

Getting Started

This is a quick setup guide, not a comprehensive write up, please proceed to the following sections for more detail.
In the main directory of the mtg audio game folder, is the executable file mtg.exe. This will launch the application. when you first open the program, it will focus on the zone tree, which is a list box that contains the zones:
hand, Creatures, Lands, Other Spells, Graveyard, Exile, and Library.
After the name of each zone is the number 0, which is the number of cards in each zone. The zone tree is probably where you will spend most of your time. If you tab over, the next element is the command input box. This texstbox is used to input commands, which is just an alternative interface to using the zone tree. However, there are some things that you can do with commands that you can not do with the zone tree. Also, This is where you would type out your search queries to search for cards. If you tab one more time, you will find the output box, which just keeps a record of everything you do for reference.

In the main directory of the program folder where you clicked on mtg.exe there is a folder called decks. This will contain all of the text files that represent your decks. To create a new deck, just create a new text file in a simple text editor like notepad on Windows, or Text Edit on Mac, and save it in your decks folder. In your new deck file, you can start writing down the name of each card that you want in your deck one per line, for example:

mountain
mountain
mountain
island
island
Storm Fleet Sprinter
etc...

To help in deck creation, a search engine has been built into the game. If you open mtg.exe, tab over to the command textbox, and type search followed by a space, you can type in the name of the card you want to lookup surrounded in quotes if the card name has spaces. The search engine will complete partially written words as long as you have the first letter or two correct.
Examples:
search negate -- finds the card with the name of negate
search “th in the i” - finds the card thing in the ice
search “storm f spri” - finds the card storm fleet sprinter
The search engine can also search by color, converted mana cost, type, set, block, watermark, etc... If you are looking for more details on the search engine, scroll down to the section on the search command. Remember that all search values are case insesetive. More complicated examples are included below:

search text=flying color=u type=angel
This command will search for all cards with flying in the card text, is blue, and is of the type angel. This gave me 11 results.

search set=“shadows over in” color=r type=wolf cmc=2 text=haste This command will search for all cards in the set of Shadows Over Inastrad, that are red, has a type of wolf, a converted mana cost of 2, and includes haste in the card text.

search set=[ixalan] color=r|w|g type=dino this search command will find all cards in the set of Ixalan, but this specifys exactly those letters, and no more and no less, so sets like rivals of ixalan will not be included in the search, which would be the case if the brackets were not used. This command also specifys that the search results must have in their casting cost either red, blue, green, or any combonation there of. This will not prevent results that are black green, red white, etc... though. Finally the type argument is provided with a value of dino, which will likely return cards with a type of dinosaur.

As you can see, the search queries can get pretty complicated, refer to the search command section for more information.

To load a deck, open the program and press CTRL+L, which will open a dialog to allow you to choose the deck you would like to use. Once you press enter, it will take a couple seconds to load and finally notify you when it is finished. You can now look at your deck in the program if you tab to the zone tree and scroll down to the bottom to your library. Press left and right errows to navigate through your cards. A short list of abreviated shortcut keys are included below. Note that a card must be selected for these keys to work.

  • 1 - speaks card mana cost
  • 2 - speaks power and toughness
  • 3 - speaks card text
  • 4- speaks card type
  • 5 - speaks card flavor text
  • V - opens a window and displays all card propertys in text, navigatable by the errow keys.
  • space - taps a card (if tapped, the word tapped is appended to the card name, if untapped, nothing is appended to the end of the name)
To shuffle, press ctrl+h, then press ctrl+shift+d to draw 7 cards, and to draw 1 card, press ctrl+d. Move back up to the top of the zone tree and and right and left errow to view the cards you drew. to move a card to the battle field, graveyard, exile, or back to the library from your hand, press:
  • enter - auto plays a card from a zone based on card type. If the card is a creature, it will move the card to the creatures zone, if land, it will be played in the lands zone, and if any other type, it will be moved to the other spells zone.
  • shift+enter - will move the card to your hand
  • ctrl+enter - moves a card to the library. Opens a menu to either move the card to the top, bottom, or other. If other is chosen, a dialog is opened to let you choose the index you would like to move your card to.
  • backspace - moves a card to your graveyard
  • shift+backspace - moves a card to exile
Finally, press CTRL+u to untap all cards on your battlefield. This should provide the basics needed to start playing. There are many other key commands and text commands that can make your experience even better. Read on to learn more.

Notes

While reading through this manual, and interpreting the errors in the game for invalid commands, the folloing style formatting is used.
Commands will always be shown in quotations, but do not include the quotations in the command when you go to type the text into the command box. All commands are parsed by spaces. The format is: the command is given, then the arguments, all seperated by spaces. For example,
“view hand”
shows the command “view” and the argument “hand”, which is a zone. Zones are areas where cards can be placed. A list of the zones and the associated commands that can be used to refference a specific zone can be found in the Zones section of this manual. A command is often represented by giving the command followed by a sequence of words all in brackets. All words wrapped in brackets are arguments, which just means that is the place where you will input the zone, card, or quantity. if an asterisk or (*) is found before an argument, it means that the argument is optional.

for example, to view a zone or card, you must use the view command
“view *[zone] *[card]”
This command has two optional arguments since there is a star in front of both arguments. Also, note that any arguments that are plural, followed by an elipsis means that 1 or more arguments can be given.

The program will ignore all extra spaces and capitalizations in a command, as long as the command and arguments are preserved. One thing to remember is that if you are typing the name of something that has spaces, you must put quotation marks around the text, since the game can only tell one argument apart from another by spaces, names can cause a problem. So, if you are wanting to view “Geier Reach Bandit” you must use a similar sintax:
‘ view hand “Geier Reach Bandit” ’ You can use either single or double quotes.

When moving cards from one location to another, weather using the move draw, mil, or any other commands that may be created to move cards, the card or cards added to the destination zone will be added to the top of the list, except in the land zone case, since the lands are auto sorted to group like names together, and the draw command, which will move a card or cards from the library to your hand, and sort by , converted mana cost, type, and then color.

The last thing to note is that cards can be referenced to by name or by index. The index is the location in the zone that the card is currently at. When you view all the cards in a zone, the index is printed next to the cardname, so instead of
“view hand Negate"”
You can type
“view hand 1”
asuming that Negate is index 1. This provides a means of accessing cards without needing to reference the card directly by name, which can be hard do to the complicated spelling of card names. Anywhere you see the argument [card], you may either provide a name, or an index to refer to that card.

Folder Structure

There are a couple folders in the main game folder that you should be aware of. The decks folder, is where you should include all of your decks. Your decks are text files that should contain a list of card names. For more, see the decks section. The other folder you should know is the database folder. in this folder, is a mtg.db file, which is the database itself that the game depends on, a folder called resources, an executable called buildDatabase, and an executable called updateDatabase. Clicking on buildDatabase will pull up a commandline interface that will be used to display the progress as the entire database is rebuilt from the ground up. This process will take several minutes. To just update the database, you can click on the updateDatabase executable, which will take significantly less time to check if there are any missing sets from the local database. There should be three files in resources, setCounter.txt, cardID.txt, and sets.txt. Do not change any of these files, they are used for createing and updating the database. the setsCounter, contains the running total number of sets in the database, and the cardID, provides the total number of cards in the database. These files are used to keep track of these to provide id's in the database tables. The sets file contains a list of all of the sets in the database. It is formatted by:
id|setName|blockName
. Do not manually edit this file, but it is useful to use to get spellings of sets or blocks.

User Interface

The user interface is comprised of a listbox that is refered by as the zone tree, and two text boxes. The zone tree is a list of all of the zones, hand, lands, creatures, other spells, graveyard, exile, and library, and when you press left or right errow on a zone, it will read the cards in that zone. The editable textbox is a single line box that takes your commands, where the second box is a read only box for all of the output text. Most of your time will be spent in the zone tree, but if you need to use the search command, or want to perform a more spacific task, then you will sometimes use the command input box.

The zone tree supports the standard keyboard navigation for listboxes, where up and down errow will navigate through each zone, home and end wil take you to the beginning and end of the list respectively, and pressing a letter will navigate to an item in the list that starts with that letter. the number of cards in the selected zone is appended to the end of the zone name for convenients. When cards are in a zone, you can navigate through each card by pressing left and right errows, and control+left errow will navigate to the first card on the far left, where control+right will navigate to the end of the card list on the far right.

When you type in a command into the command input box the currently running screenreader will speak the output while your focus is still in the command input box. If you tab, all aditional commands except for the first will dump the cursor at the beginning of the new output text. from the output box, you can start typeing with out having to tab, and it will change focus for you and immediately start typing characters into the command input box. If you press up or down errow, you can cycle through up to your last 50 commands you have given. Press up errow to get your previous commands, and down errow to cycle back through those commands all the way down to a blank input box.

Deck

The deck is constructed in a text file in the folder “decks” folder, where the exicutable is located. You may create as many files as you wish in this folder and call your files what ever you want, however, you may only load text files. Programs such as notepad on Windows, or text edit on mac are used for this purpose. You can save a text file in microsoft word, but it is possible you may get strange behavior because microsoft word may include aditional characters that are not visible that could cause problems. In your text file, you need to include the exact spelling of all of the cards you would like in your deck seperated by newlines. The program will help a little, all capitalization will be ignored, otherwise your spelling and grammar must be perfect to insure that the card you want is the first result in the search for that card name. It takes a couple seconds to load the deck when a deck is loaded, so be patient. This is an issue that will hopefully be addressed in the future.

To insert comments into the deck files, use two forward slashes //, to comment out a single line, and /* */ for a multi line comment. these are C style comments, so any programmers should be familiar with this comment style. Examples are below.

negate
thing in the ice
//this is a single line comment that will not be read as a card name.
grasp of darkness
/*
this is a multi line comment.
All text between these symbols will be ignored. */

You may want to use comments for organizing your deck file, or for using a side board.

Zones

The zones in this game are all of the locations that cards can be moved to. You have the freedom to move cards from one zone to another using the provided commands below. I have provided variants on the zone names, to make it faster to type.

Note: it is recommended to use the zones for there purpose since there are some reinforcements of card placement. This just means, don't use the creature zone for your graveyard, your exile zone for your deck, or any other strange behavior like that. If you do, a lot of the commands, such as play, discard, mill, attack, etc... are performing actions specific to a zone with the asumption that you are using that zone for its intended purpose, so it will perform those actions accordingly and mess up your board.

When moving cards from one zone to another, it is set to clear all status properties before adding the card into the new zone. This means that tapped cards are untapped, transformed cards are flipped, etc...

The zones are: library, hand, exile, graveyard, creatures, lands, and otherSpells. The variants to these commands are listed below.

Zone Description
library, lib This is the zone where your library is stored, when you first load a deck, the deck is loaded into this zone.
exile, exl This is where you should put your exiled cards
graveyard, grave, grv, gr This is where you should put cards that are supposed to go in your graveyard.
hand, hnd This is the zone where you keep the cards that are in your hand
creatures, creature, crt, cr This zone is used to keep your creatures that are on the battlefield.
lands, land, lnd This zone is used to store your lands. This zone auto sorts the cards here so that lands with the same name are grouped together.
otherSpells, other, othr, oth This zone is used for all other spells that are not accounted for.

Searching the Database

This is used to search the local database that is packaged with this game. You do not need internet to run this command.
note: All given search values ignore capitalization, and if your property has spaces,and you did not complete the words, the program will try to find all matches with the given string putting variables at the end and everywhere there is a space. This will insure that a search for the card “geier reach bandit” will also work if given “geie rea bandit”.
the first argument is a cardName, if you are searching for a spacific card by card name, you must include it directly after the search command. You can include other search parameters after the card name if you wish.
The order of the properties does not matter, you can provide them in any order you wish.

The card properties are the same as those used to sort with minus the card name.
Properties to search by:
(manaCost, mc, color), (convertedManaCost, cmc), type, text, power, toughness, (watermark, water, wm), set, block

the card property must be directly followed by an equals sign to provide the value for that property.
“search “mardu” set=“Khans of Tarkir””
This will search for all cards in the Khans of Tarkir set for cards that have “mardu” somewhere in the card name regardless of capitalization. Note that you can not put a space before or after the = (equals) sign because it will treat the equals sign and the surrounding text all as seperat arguments if you do, and it will throw an error.

Note that you can give multiple flags to help your search where it makes sence. For example, you could give a text flag saying draw 1 and a second text flag for flying, with a type of creature. This will search for the keywords draw 1, somewhere in the card text, and for flying somewhere in the card text too for all cards of type creature.
This is helpful because if you just said
“draw 1, flying”
The program will look for all insinces in a card's text where it says draw 1, flying, which is probably none. So, you can look for many different keywords that could be located anywhere in the card text by using multiple text flags.
For example:
“search text=“draw 1” text=“flying” type=“creature””
You can also search for hybrid colored cards with multiple color flags. For example:
“search color=u color=r type=“instant””
will find all blue red cards that are instances.
You must remember that setting multiple flags is using the “and” operator, so using multiple color flags or text flags makes sence, but using multiple set or block flags does not since a card is only ever apart of one set. This is technically not true, however if a card is from multiple sets, more than one instance of that card is in the database, so a card is not actually apart of more than one set or block at the same time. Perhaps this will be addressed in the future, but it is not a priority.

Only 400 results can be displayed at a time, so if your search results exceed 400, the results to be displayed will be truncated down to 400. This limitation is due to wx python's text control wigit only allowing so many characters in the text box at any given time. This is an issue that is not a priority to fix since I feel that few people will ever want to look through more than 400 cards at any given time.

Aditional search sintax for better results:
brackets [] - used for identifying exact text that you don't want to be misconstrued. For example, if you search for set=ixalan, it will return back cards from the Ixalan set, as well as rivals of ixalan. To avoid this, you can wrap your value ixalan in brackets so that it will search for exact text and not place wild card variables at the end of each word. Note that you can not wrap individual words in brackets, only entire values. If you try to wrap a word in brackets you will get an error or no results. Also note that capitalization still does not matter, only spelling.
Example:
“search set=“[ixalan]””
or operator | - you can use the | (verticle bar) character, shift+backslash, to specify an or operation. So, if you want to search for multiple sets at the same time, type ixalan|shadows over innistrad, and it wil return all cards from ixalan or shadows over innistrad. The or operator can be used with any card property, such as block, color, etc... You can still use brackets around exact strings.
example:
“search block=“[ixalan]|shadows over innistrad” type=creature color=u|r cmc=1|2|3”
This query will find all cards that are in the ixalan block or the shadows over innistrad block that are red or blue creatures with a converted mana cost of 1, 2, or 3.

It is also possible to search for hybrid colors using 1 color flag. just say something like color=urb, which will search for red, blue, or black cards. The order of colors does not matter.
example:
“search set="shadows over innistrad" color=rb type=creature”
This will search for all red black creatures in the Shadows over Innistrad set.

Running with out an Exicutable

If you don't plan on running this game with out an exicutable, you can skip this section.

The game was constructed in python 3.4, and depends on accessible_output2 and wx python. These libraries may have other unmentioned dependencies that may need to be installed to get these libraries to work.

Installations:
You must install python 3.4, which you can go to

www.python.org

to find the python 3.4 download. I was unable to get accessible_output2 working with higher versions of python, so we are stuck for now at python 3.4.

the next thing to install is pywin32 for python 3.4. this is a requirement for accessible_output2. The other two dependencies for accessible_output2 are already included in the main directory of the program: libloader, and platform_utils

The last step is to install wx python. The easiest way is to use a program built into python called pip, which is used to install libraries. Open a command prompt: press windows+R type cmd into the run dialog and press enter once the command prompt opens, type, or copy and paste the command: python -m pip install wxpython and press enter Note: if you have other versions of python running already, python may not install wx python to python 3.4. Optionally, you can download wx python as an exicutable, but it is not recommended since pip insures you get the correct version.

Finally, to run the game, navigate to the main folder for the game, and press enter on mtg.py. This should open the game if you did not have any previous versions of python installed before, or if you did not change how python files are opened. If the game is not opened, open a command prompt window in that directory by pressing alt+d in that directory in file explorer and type cmd and enter. then type: python mtg.py again, if you have other python versions installed, this may not work. Consult the python manual on how to deal with installing multiple versions of python at the same time.
